**In this page**, I will share my entire 13-week AWS Cloud internship journey, from learning foundational knowledge to implementing the **Serverless Student Management System** in a real-world project model.

Throughout the learning process, I went through:

- Weeks 1–7: Learning foundational AWS knowledge (Compute, Storage, Networking, Security, Monitoring…)
- Week 8: Mid-term exam and project topic selection
- Weeks 9–13: Building the Backend for the Serverless Student Management System

Below is the summary of weekly work:

**Week 1:** [Onboarding, AWS basics, IAM](1.1-week1/)

**Week 2:** [Security (KMS, Secrets Manager), VPC, EC2](1.2-week2/)

**Week 5:** [Serverless (Lambda, API Gateway), ECS/Fargate, CloudWatch](1.5-week5/)

**Week 7:** [Mid-term review](1.7-week7/)

**Week 9:** [Architecture design, DynamoDB, Cognito, Authentication APIs](1.9-week9/)

**Week 10:** [APIs (Classes, Subjects), Lambda deployment, API Gateway](1.10-week10/)

**Week 11:** [CloudWatch monitoring, Notifications, Optimization](1.11-week11/)

**Week 12:** [WAF, GuardDuty, CI/CD Pipeline, CloudFront](1.12-week12/)

**Week 13:** [Testing, Cost Optimization, Documentation, Presentation](1.13-week13/)
